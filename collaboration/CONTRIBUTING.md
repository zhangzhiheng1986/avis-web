# Avis 系统贡献指南

> 版本：V1.0 | 最后更新：2026-06-01

---

## 1. 欢迎贡献

Avis 系统致力于成为全球航空航天阀门领域最权威的开放知识平台。无论您是阀门设计工程师、材料科学家、系统工程师，还是航天爱好者，您的每一次贡献都将推动这一知识体系的演进。

---

## 2. 贡献者类型与权限

| 角色 | 权限范围 | 获取方式 |
|------|----------|----------|
| **读者** (Reader) | 只读访问知识库 | 默认 |
| **贡献者** (Contributor) | 提交 RFC 提案、内容编写 | 首次成功提交 RFC |
| **评审者** (Reviewer) | 技术评审、质量把关 | 持续高质量贡献 + 邀请 |
| **维护者** (Maintainer) | 合并权限、版本发布 | 核心贡献 + 任命 |

---

## 3. RFC 提案流程

### 3.1 何时提交 RFC

以下情况 **必须** 走 RFC 流程：
- 新增章节或重大内容扩充（>500字新增内容）
- 修改现有章节的核心结论或技术判断
- 调整知识体系结构（编章重组）
- 修改术语定义或分类体系

以下情况可 **直接提交** 修正：
- 错别字、标点修正
- 格式统一化调整
- 引用编号修正
- 死链修复

### 3.2 RFC 文档规范

使用 `collaboration/templates/rfc-template.md` 模板，必须包含：

```
# RFC-XXXX: [提案标题]

## 元数据
- 提案人：
- 日期：
- 状态：草案 | 评审中 | 已接受 | 已拒绝
- 关联章节：

## 动机
为什么需要此变更？

## 详细设计
变更的具体内容。

## 影响评估
对现有知识体系的影响分析。

## 引用来源
变更所依据的权威文献（至少 C 级以上）。

## 评审记录
（评审过程中追加）
```

### 3.3 评审标准

| 维度 | 标准 | 权重 |
|------|------|------|
| **技术准确性** | 内容是否符合工程科学原理，有权威引用支撑 | 40% |
| **体系一致性** | 是否与现有知识体系自洽，不引入矛盾 | 30% |
| **覆盖完整性** | 是否填补了已知的知识空白 | 20% |
| **表达清晰性** | 表述是否清晰、逻辑是否严密 | 10% |

## 4. 内容编写规范

### 4.1 Markdown 标准

- 使用 GFM（GitHub Flavored Markdown）标准
- 文件编码：UTF-8（无 BOM）
- 行尾符：LF（Unix 风格，`\n`）
- 标题层级不超过 H4（`####`）

### 4.2 章节结构模板

```markdown
# 第X章 章节标题

> 本章摘要：一句话概括本章核心内容。

---

## X.1 概述
## X.2 核心原理
## X.3 工程实践
## X.4 前沿进展
## X.5 小结

---

> 参考文献：级别-编号
```

### 4.3 公式规范

- 使用 LaTeX 语法：`$inline$` 或 `$$block$$`
- 公式编号格式：`(式 X.Y-Z)`，X为章号，Y为节号，Z为节内序号

### 4.4 引用格式

- 文内引用：`[A-001]`、`[B-042]` 格式
- 引用级别：A（权威标准）、B（教材专著）、C（学术论文）、D（工业报告）、E（工程经验）
- 所有引用必须在第五编参考文献章节中有对应条目

### 4.5 表格规范

- 表格必须有序号和标题：`表 X.Y 表格标题`
- 推荐使用标准 Markdown 表格语法

## 5. 版本编号规则

### 5.1 版本号格式

`MAJOR.MINOR.PATCH`

- **MAJOR**：知识体系结构重大调整（编章重组、分类体系变更）
- **MINOR**：新增章节（≥1章）或重大内容扩充（≥5000字新增）
- **PATCH**：勘误修正、格式调整、引用更新、小幅内容补充（<5000字）

### 5.2 发布节奏

- PATCH 发布：按需，每周可能多次
- MINOR 发布：每月至每季度
- MAJOR 发布：每半年至一年

---

## 6. 行为准则

- **尊重工程事实**：所有技术论断必须有引用支撑，拒绝无依据的个人观点
- **建设性评审**：评审聚焦于技术内容，不针对个人
- **承认贡献**：所有合并的 RFC 提案人列入贡献者名单
- **知识共享**：本手册以开放知识许可发布，贡献即同意共享
