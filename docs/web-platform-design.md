# Avis Web平台架构设计文档

> 版本：V1.0 | 日期：2026-06-01 | 作者：小贤
>
> 将65章Markdown手册 + Git版本控制的Avis系统，升级为全球可访问的Web可视化知识平台。

---

## 1. 总体定位

### 1.1 Avis Web平台 = 全球航空航天阀门领域知识体系的"活的大厦"

Avis Web平台不是一本静态电子书，而是一座**持续生长、可交互、可协作的知识大厦**。它承载着约60万汉字的航空航天阀门研发设计知识体系，面向全球工程技术人员、研究人员和学生开放。

### 1.2 三个核心目标

| 目标 | 描述 | 衡量标准 |
|------|------|----------|
| **浏览学习** | 提供结构化、可导航的完整知识体系，支持顺序阅读与跳转查阅 | 完整的五编结构目录树、章节间交叉引用 |
| **知识检索** | 全文搜索、术语联动、知识图谱可视化，让知识触手可及 | 毫秒级搜索响应、术语一键跳转 |
| **协同共创** | 开放RFC提案、社区评审、版本管理，让全球专家共同完善知识体系 | GitHub PR流程、版本发布节奏、贡献者网络 |

### 1.3 设计原则

- **内容即代码（Docs as Code）**：Markdown源文件 → Git版本控制 → 自动化构建 → 静态站点部署
- **渐进增强（Progressive Enhancement）**：Phase 1静态站 → Phase 2交互增强 → Phase 3共创平台 → Phase 4智能智库
- **零运维成本（Serverless）**：纯静态站点 + 免费CDN + 无服务器评论系统
- **国际化就绪（i18n Ready）**：中文主站 + 架构预留多语言支持

## 2. 技术架构

### 2.1 推荐技术栈

```
┌─────────────────────────────────────────────────────┐
│                    呈现层（CDN）                       │
│         GitHub Pages / Cloudflare Pages             │
├─────────────────────────────────────────────────────┤
│                  构建层（SSG）                         │
│     VitePress (Vue 3 + Vite + Markdown)             │
├─────────────────────────────────────────────────────┤
│                  交互层（浏览器端）                     │
│  ┌──────────┐ ┌──────────┐ ┌──────────────────┐    │
│  │ 全文搜索  │ │ 知识图谱  │ │ 评论系统 (Giscus) │    │
│  │ (内置)   │ │ (D3.js)  │ │ (GitHub Discuss) │    │
│  └──────────┘ └──────────┘ └──────────────────┘    │
├─────────────────────────────────────────────────────┤
│                  内容层（源文件）                       │
│     Markdown (.md) + Git (版本控制)                  │
├─────────────────────────────────────────────────────┤
│                  协作层（GitHub）                      │
│     Issues → RFC Proposals → PR → Review → Merge    │
└─────────────────────────────────────────────────────┘
```

### 2.2 各组件选型理由

#### 2.2.1 静态站点生成器：VitePress

| 对比维度 | VitePress | Docusaurus | Hugo | MkDocs |
|----------|-----------|------------|------|--------|
| 构建速度 | 极快（Vite） | 较快 | 极快 | 快 |
| Markdown支持 | 原生 | 原生 | 原生 | 原生 |
| Vue组件 | 内置 | React | 无 | 无 |
| 全文搜索 | 内置（MiniSearch） | 需Algolia | 无内置 | 需插件 |
| 中文生态 | 优秀 | 良好 | 一般 | 良好 |
| 学习曲线 | 低 | 中 | 低 | 低 |
| 自定义主题 | 简单 | 较复杂 | 复杂 | 有限 |

**选型结论**：VitePress在中文技术文档领域有广泛使用（Vue.js官方文档、Vite官方文档均使用VitePress），构建速度极快，内置全文搜索无需第三方服务，Vue3组件支持为后续知识图谱等交互功能提供扩展基础。

#### 2.2.2 托管平台：GitHub Pages / Cloudflare Pages

| 对比维度 | GitHub Pages | Cloudflare Pages |
|----------|-------------|------------------|
| 免费额度 | 100GB带宽/月 | 无限带宽 |
| 全球CDN | Fastly | Cloudflare（节点更多） |
| 构建集成 | GitHub Actions | 内置CI/CD |
| 自定义域名 | 支持（免费HTTPS） | 支持（免费HTTPS） |
| 部署速度 | 较快 | 极快 |

**选型结论**：首选GitHub Pages（与GitHub生态深度集成，PR预览方便）；备选Cloudflare Pages（带宽更大、节点更多）。

#### 2.2.3 搜索方案：VitePress内置本地搜索

VitePress内置基于MiniSearch的本地搜索，特点：
- **零配置**：开箱即用，无需注册第三方服务
- **纯前端**：搜索索引在构建时生成，搜索过程完全在浏览器端完成
- **中文友好**：MiniSearch支持中文分词
- **离线可用**：静态资源加载后即可搜索

> 不选择Algolia的原因：①免费额度有限（1万次/月）；②需要爬虫收录；③对非开源项目收费。Avis当前阶段不需要第三方搜索服务。

#### 2.2.4 评论/讨论系统：Giscus

Giscus是一个基于GitHub Discussions的评论系统：
- **无服务器**：评论数据存储在GitHub Discussions中
- **免费**：利用GitHub Discussions的免费额度
- **权限控制**：评论者需要GitHub账号，天然防垃圾
- **通知集成**：新评论自动通过GitHub通知提醒维护者

#### 2.2.5 版本管理：Git + Git Tag

```
Git工作流（基于Trunk-Based Development）：

main ───●─────●─────●─────●─────●
         │     │     │     │     │
        V5.0  V5.1  V5.2  V6.0  V6.1  ← Git Tags（触发自动部署）
         │           │
   feature/rfc-xxx  fix/typo-xxx        ← 短期分支
```

- 每个版本对应一个Git Tag
- Tag推送自动触发GitHub Actions构建部署
- 版本号遵循语义化版本规范（SemVer）

## 3. 四阶段演化路线

### 3.1 Phase 1 — 静态文档站（当前阶段，1-2周）

**目标**：将65章Markdown手册转化为可浏览、可搜索的Web文档站。

**交付物**：
- VitePress站点搭建完成
- 五编结构侧边栏导航
- 所有章节内容可浏览
- 全文搜索功能可用
- 部署到GitHub Pages
- 响应式设计（桌面+移动端）

**技术要点**：
- VitePress默认主题 + 自定义品牌色
- 侧边栏按五编+附录自动生成
- 启用`search: { provider: 'local' }`
- 配置`lastUpdated: true`显示最后更新时间

### 3.2 Phase 2 — 增强交互（2-4周）

**目标**：从纯文档站升级为知识探索平台。

**交付物**：
- **知识图谱可视化**：使用D3.js或Mermaid展示章节间引用关系、概念层级
- **术语联动**：点击术语自动弹出定义卡片，术语间建立超链接网络
- **全局搜索增强**：支持搜索建议、模糊匹配、搜索结果高亮
- **阅读进度与书签**：浏览器localStorage保存阅读位置

### 3.3 Phase 3 — 共创平台（1-3个月）

**目标**：开放全球协作，建立社区驱动的知识演化机制。

**交付物**：
- **在线RFC提案系统**：基于GitHub Issues + 自定义表单的结构化提案流程
- **版本对比视图**：任意两个版本间的Diff高亮展示
- **同行评审面板**：评审意见直接在章节侧边展示
- **贡献者排行榜**：基于Git提交统计的贡献者网络图
- **变更日志自动生成**：从Git提交历史自动生成CHANGELOG

### 3.4 Phase 4 — 智能智库（3-6个月）

**目标**：引入AI能力，打造个性化学习与研发辅助平台。

**交付物**：
- **AI智能问答**：基于RAG（检索增强生成）的领域问答机器人
- **个性化学习路径**：根据用户角色（学生/设计师/测试工程师）推荐阅读路径
- **知识卡片生成**：AI自动从章节中提取关键概念生成闪卡
- **API接口**：开放RESTful API，供第三方工具集成Avis知识库
- **多语言翻译**：AI辅助核心章节英文翻译

## 5. 协作工作流

### 5.1 贡献者角色

| 角色 | 权限 | 职责 |
|------|------|------|
| **浏览者（Visitor）** | 只读 | 浏览学习、发现错误通过Issue反馈 |
| **贡献者（Contributor）** | Fork → PR | 提交改进提案、修复错误、翻译章节 |
| **评审者（Reviewer）** | PR Review | 审核技术内容准确性、格式规范性 |
| **维护者（Maintainer）** | Merge权限 | 合并PR、发布版本、管理Issue |
| **作者（Author）** | 所有权限 | 最终决策、版本规划、质量把关 |

### 5.2 RFC提案流程

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│ 发现问题  │───→│ 创建RFC   │───→│ 社区讨论  │───→│ 核心团队  │───→│ PR合并   │
│ 或改进点  │    │ 提案Issue │    │ (≥2周)   │    │ 评审      │    │ 版本发布  │
└──────────┘    └──────────┘    └──────────┘    └──────────┘    └──────────┘
```

**RFC提案模板要素**：
- 提案标题（简洁描述改进内容）
- 动机与背景（为什么要改）
- 详细方案（具体怎么改）
- 影响范围（涉及哪些章节）
- 参考依据（引用的标准、文献）
- 提案人信息

### 5.3 版本发布节奏

| 版本 | 周期 | 内容 |
|------|------|------|
| **修订版（V5.x）** | 每月 | 勘误修正、小改进、格式优化 |
| **次版本（Vx.0）** | 每季度 | 新增章节、重大修订、功能升级 |
| **主版本（Vx.0.0）** | 每年 | 知识体系重构、平台架构升级 |

### 5.4 质量保证环

```
编写/修改 → 自我检查 → 同行评审 → 自动化检查 → 合并 → 发布
              ↑                                              ↓
              └──────── 用户反馈（Issue） ←──────────────────┘
```

自动化检查项：
- Markdown语法检查（markdownlint）
- 链接有效性验证
- 术语一致性检查
- 构建成功验证

## 6. 国际化策略

### 6.1 当前阶段：中文主站

- 全部内容以简体中文呈现
- 关键术语附英文对照（如"阀门/Valve"）
- URL结构预留语言前缀（如`/zh/`）

### 6.2 中期目标：核心章节英文翻译

- 优先翻译：第1章（概论）、第6-9章（基础设计）、第32章（可靠性）、第51章（术语表）
- 翻译策略：AI初译 + 领域专家审校
- 中英文双版本并行，通过语言切换器切换

### 6.3 长期愿景：多语言支持

- VitePress原生支持多语言（`locales`配置）
- 社区驱动的众包翻译
- 翻译记忆库（TM）维护术语一致性
- 目标语言：英文、法文、日文、俄文
