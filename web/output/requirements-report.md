## 1. 引言与背景

### 1.1 项目背景

Avis 知识网络页面是一个基于 D3.js 力导向图的可视化工具，用于展示《航空电子系统综合化设计》教材 62 个章节之间的知识关联关系（84 条边）。页面作为 VitePress 文档站点的独立子页面运行，承担着帮助读者理解知识体系拓扑结构、发现前置依赖和交叉引用关系的核心职能。

### 1.2 现状评估

根据 2026-06-03 的系统审计报告，当前知识网络页面存在 20 项问题，综合评分 5.6/10。可视化内核质量较高（8.5/10），但生产化准备不足——在可靠性（CDN 单点故障）、可访问性（ARIA 完全缺失）、响应式设计（无移动端适配）三个维度存在严重短板。

### 1.3 需求论证范围

本文档基于审计报告中的 20 项问题，结合知识网络可视化领域的工程最佳实践，重新论证功能需求和非功能需求，为后续架构重构提供需求基线。

## 2. 用户故事与使用场景

### 2.1 核心用户画像

| 角色 | 描述 | 核心诉求 |
|------|------|---------|
| **学习者** | 正在学习航空电子系统综合化设计的读者 | 理解章节间的依赖关系，规划学习路径 |
| **教师/培训师** | 使用教材进行课程设计的教学人员 | 快速定位某一知识板块的覆盖范围 |
| **视障用户** | 依赖屏幕阅读器访问的读者 | 通过键盘和辅助技术浏览知识结构 |
| **移动端用户** | 在手机或平板端访问的读者 | 在小屏上浏览和搜索知识网络 |
| **内容维护者** | 负责更新教材和节点数据的编辑 | 同步侧边栏配置与知识网络数据 |

### 2.2 核心使用场景

| 编号 | 场景名称 | 用户故事 | 触发条件 | 验收标准 |
|------|---------|---------|---------|---------|
| US-01 | 全局知识浏览 | 作为学习者，我希望在打开页面时看到完整的知识网络图，以便对全书结构建立整体认知 | 首次加载页面 | 62 个节点、84 条边在 3 秒内完成渲染并稳定显示 |
| US-02 | 前置依赖探索 | 作为学习者，我希望点击某个章节节点时看到其前置依赖和后续关联，以便规划学习顺序 | 点击节点 | 高亮显示与该节点直接关联的边和目标节点，侧边栏显示节点详情 |
| US-03 | 语义搜索定位 | 作为学习者，我希望通过关键词搜索找到相关章节，以便快速定位知识点 | 输入搜索词 | 匹配结果以视觉差异（高亮/变暗）呈现；无匹配时显示文字反馈 |
| US-04 | 聚类模式浏览 | 作为教师，我希望通过聚类视图按"编"级别浏览知识板块，以便了解各知识域的覆盖范围 | 缩小视图至阈值 | 6 个聚类圆按编分布，圆大小反映各编节点数量 |
| US-05 | 移动端查阅 | 作为移动端用户，我希望在手机上浏览知识网络，以便随时复习知识关联 | 小屏设备访问 | 侧边栏折叠为可展开抽屉，力导向图占据主视口 |
| US-06 | 屏幕阅读器访问 | 作为视障用户，我希望通过屏幕阅读器和键盘导航了解知识结构 | 启用辅助技术 | 每个节点可被 Tab 键聚焦并朗读标签/描述 |
| US-07 | 暗色模式切换 | 作为夜间阅读用户，我希望切换到暗色模式，以便减少视觉疲劳 | 点击暗色模式按钮 | 所有 SVG 节点、边、文本颜色平滑过渡到暗色主题 |
| US-08 | 章节同步导航 | 作为 VitePress 站点用户，我希望知识网络页面与侧边栏导航数据一致，以便无缝跳转 | 对比侧边栏与图谱 | 图谱节点与侧边栏章节一一对应 |
| US-09 | 离线可用性 | 作为网络不稳定环境下的用户，我希望 CDN 故障时页面仍能正常工作，以便不中断学习 | CDN 不可达 | 页面正常渲染，力导向图可用 |
| US-10 | 窗口缩放适配 | 作为多显示器用户，我希望在不同窗口大小下知识网络都能正常显示，以便灵活工作 | 拖拽窗口边缘 | 分辨率从 375px 到 2560px 均正常渲染，无溢出或截断 |

## 3. 功能需求（FR）

### 3.1 数据管理

| 编号 | 需求描述 | 来源审计项 | 优先级 |
|------|---------|-----------|--------|
| FR-01 | 所有统计数据（节点数、边数）必须从 JSON 数据源**动态读取**，页面中不得存在硬编码数值 | H-1 | P0 |
| FR-02 | JSON 数据文件必须是颜色/样式定义的**唯一权威源**，D3 代码直接从 JSON 读取样式配置，不得在 JS 中重复定义 | M-1 | P1 |
| FR-03 | 知识网络节点集必须与 VitePress 侧边栏配置 (`config.mjs`) 中的章节条目**保持一致**，新增/删除章节应触发双向校验 | H-2 | P1 |
| FR-04 | 数据加载失败时，页面必须显示用户可理解的错误提示，而非白屏 | C-1 | P0 |

### 3.2 可视化渲染

| 编号 | 需求描述 | 来源审计项 | 优先级 |
|------|---------|-----------|--------|
| FR-05 | 力导向仿真在聚类模式下必须**暂停计算**，退出聚类模式后以 `alpha(0.3).restart()` 恢复 | M-3 | P1 |
| FR-06 | 进场淡入动画必须与 simulation 稳定状态同步，以 `simulation.on('end')` 事件为触发条件 | M-6 | P2 |
| FR-07 | 聚类圆半径必须与对应编的节点数量**正相关**，公式: `r = max(18, 10 + count * 3.5)` | L-3 | P2 |
| FR-08 | 暗色模式切换时，SVG 节点填充色和边描边色必须通过 CSS transition 实现**平滑过渡**（≥ 300ms） | L-5 | P2 |
| FR-09 | 信息面板内容更新必须以 `transitionend` 事件为触发条件，禁止使用 `setTimeout` 硬编码延迟 | L-4 | P2 |

### 3.3 搜索与交互

| 编号 | 需求描述 | 来源审计项 | 优先级 |
|------|---------|-----------|--------|
| FR-10 | 搜索无匹配结果时，必须在信息面板显示明确的"未找到匹配章节"文本反馈 | M-5 | P1 |
| FR-11 | 聚类模式与详情模式切换期间，统计数字必须保持**原子性**，不得出现短暂错误值闪烁 | M-8 | P2 |
| FR-12 | `exitClusterMode()` 中 `isClustered` 标志位切换必须在所有 CSS transition 完成之后执行，使用 `.on('end')` 回调 | H-3 | P0 |

### 3.4 可访问性

| 编号 | 需求描述 | 来源审计项 | 优先级 |
|------|---------|-----------|--------|
| FR-13 | 所有 SVG 图形元素必须配备完整的 ARIA 标签体系（`role="img"` + `aria-label`，节点 `<title>` 子元素） | H-4 | P1 |
| FR-14 | 键盘导航必须支持 Tab 在节点间切换聚焦、Enter/Space 选中节点、Escape 退出聚类模式 | H-4 | P1 |

### 3.5 SEO 与元数据

| 编号 | 需求描述 | 来源审计项 | 优先级 |
|------|---------|-----------|--------|
| FR-15 | 页面必须包含 `<meta name="description">` 和 Open Graph 标签（`og:title`、`og:description`、`og:image`、`og:type`） | L-1 | P2 |
| FR-16 | 知识网络文档（`knowledge-graph.md`）必须创建并包含图谱使用说明、图例和数据维护指南 | L-6 | P2 |

## 4. 非功能需求（NFR）与质量属性

### 4.1 可靠性 (Reliability)

| 编号 | 需求描述 | 量化目标 | 测量方法 | 来源审计项 |
|------|---------|---------|---------|-----------|
| NFR-01 | CDN 资源加载失败时，必须自动回退到本地 bundled 版本，页面核心功能（力导向图渲染、搜索、聚类）不受影响 | CDN 故障恢复率 = 100%（本地 fallback 必须在 500ms 内激活） | 模拟 CDN 不可达，验证页面白屏时间 ≤ 0s | C-1 |
| NFR-02 | 数据加载异常（JSON 解析失败、HTTP 4xx/5xx）必须降级为错误提示页，不得静默失败或白屏 | 所有异常路径必须有用户可感知的 UI 反馈 | 模拟 JSON 404 / 格式错误，验证错误提示出现 | FR-04 |

### 4.2 性能 (Performance)

| 编号 | 需求描述 | 量化目标 | 测量方法 | 来源审计项 |
|------|---------|---------|---------|-----------|
| NFR-03 | 首次内容渲染（FCP）必须在 CDN 正常时 ≤ 2.0s，本地 fallback 时 ≤ 3.5s | FCP ≤ 2.0s (P95) | Lighthouse Performance 审计 | C-1 |
| NFR-04 | 力导向仿真在聚类模式下 CPU 占用须降低 ≥ 60%（通过暂停 simulation 实现） | 聚类模式 CPU idle ≥ 80% | Chrome DevTools Performance 面板 | M-3 |
| NFR-05 | 窗口 resize 事件处理频率 ≤ 6.7 Hz（150ms debounce），避免连续触发导致 UI 卡顿 | resize handler 调用频率 ≤ 7 次/秒 | 计数器 + Performance Observer | M-4 |
| NFR-06 | 力导向图交互帧率（拖拽/缩放/搜索高亮）必须保持 ≥ 30 FPS | 交互期间平均 FPS ≥ 30 | Chrome DevTools FPS 面板 | M-3, M-6 |
| NFR-07 | 资源总大小（HTML + JS + JSON + D3.js）≤ 500KB（gzip 后），便于移动网络加载 | gzip 后总传输 ≤ 500KB | Webpack Bundle Analyzer | — |

### 4.3 可维护性 (Maintainability)

| 编号 | 需求描述 | 量化目标 | 测量方法 | 来源审计项 |
|------|---------|---------|---------|-----------|
| NFR-08 | 样式/颜色定义源必须唯一，JSON 与 JS 不得存在重复定义 | 样式定义重复度为 0（无重复定义） | 自动化脚本扫描 | M-1 |
| NFR-09 | 代码中不得存在永远不命中的冗余分支（如 `typeof e.source === 'object'` 因调用上下文保证 e.source 始终为字符串而永不触发） | 冗余死代码覆盖率 = 0% | ESLint `no-constant-condition` + 人工审查 | M-2 |
| NFR-10 | 数据模型校验必须在页面加载时完成，JSON 加载后立即执行 schema 校验（node-edge 一致性、必填字段完整性），异常数据不进渲染管线 | 校验覆盖度 100%（所有节点/边必须通过 schema 检查） | 自动化校验脚本 | H-1, H-2 |

### 4.4 可访问性 (Accessibility)

| 编号 | 需求描述 | 量化目标 | 测量方法 | 来源审计项 |
|------|---------|---------|---------|-----------|
| NFR-11 | 页面必须通过 WCAG 2.1 Level AA 合规审计 | axe-core 扫描 0 个 Violation（排除第三方库误报） | axe DevTools 自动化扫描 | H-4 |
| NFR-12 | 所有可交互元素（节点、搜索框、按钮）必须支持键盘操作，Tab 序符合视觉布局逻辑 | 键盘即可完成所有交互操作，无需鼠标 | 人工键盘导航测试 | H-4, FR-14 |
| NFR-13 | 颜色对比度必须满足 WCAG AA 标准（正常文本 ≥ 4.5:1，大文本 ≥ 3:1） | 所有文本-背景对比度 ≥ 4.5:1 | axe DevTools / Contrast Checker | H-4 |

### 4.5 响应式设计 (Responsiveness)

| 编号 | 需求描述 | 量化目标 | 测量方法 | 来源审计项 |
|------|---------|---------|---------|-----------|
| NFR-14 | 页面必须在 ≥ 4 个断点下正常渲染，无水平滚动、无内容溢出、无功能丢失 | 375px / 768px / 1024px / 1440px / 1920px 全通过 | BrowserStack / Chrome DevTools 设备模拟 | H-5 |
| NFR-15 | 移动端（< 768px）侧边栏必须以可折叠面板/底部抽屉形式呈现，不抢占主画布空间 | 375px 视口下画布可用宽度 ≥ 屏幕宽度的 95% | 人工测量 | H-5 |

### 4.6 安全性 (Security)

| 编号 | 需求描述 | 量化目标 | 测量方法 | 来源审计项 |
|------|---------|---------|---------|-----------|
| NFR-16 | 所有外部资源（CDN D3.js）必须附带 `integrity` 哈希（SRI）进行子资源完整性校验 | SRI 覆盖度 = 100%（所有外部脚本均有 integrity） | 自动化扫描 | C-1 |
| NFR-17 | 页面不得使用 `eval()`、`innerHTML` 直接注入用户可控内容（如搜索词），防范 XSS | CSP 允许 `script-src 'self'` + CDN + SRI，无 unsafe-eval | CSP Evaluator | — |
